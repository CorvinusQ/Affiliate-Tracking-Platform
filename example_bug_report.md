# Bug Report: Ошибка 400 при фильтрации отчета по дате

**ID:** BUG-2025-001  
**Статус:** Closed  
**Приоритет:** High  
**Серьезность:** Major  
**Модуль:** Reports / Filtering  
**Окружение:** Stage, Chrome 120, macOS

---

## Описание
При фильтрации отчета по дате с выбором периода "Вчера" и часовом поясе GMT+3 API возвращает ошибку 400 Bad Request, данные не загружаются.

---

## Шаги воспроизведения
1. Авторизоваться в системе
2. Перейти в раздел "Отчеты" → `/reports`
3. Открыть фильтр дат
4. Выбрать пресет "Вчера"
5. В выпадающем списке часовых поясов выбрать "GMT+3 (Москва)"
6. Нажать кнопку "Применить"

---

## Фактический результат
- Запрос к API: `GET /api/reports?from=undefined&to=undefined&timezone=GMT%2B3`
- Ответ сервера: `400 Bad Request`
- Сообщение: `"Invalid date format"`
- На экране: пустой список, индикатор загрузки исчезает

---

## Ожидаемый результат
- Запрос к API с корректными датами: `GET /api/reports?from=2025-01-06&to=2025-01-06&timezone=GMT%2B3`
- Ответ сервера: `200 OK` с данными
- На экране: таблица с метриками за выбранный период
